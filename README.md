# Security Policy Development

## Overview

This repository contains my Week 4 internship assignment completed during the Sohail Smart Solutions Summer Training Program 2026.

The objective of this assignment was to develop governance documents that address the security gaps identified during a previous ISO/IEC 27001:2022 Annex A gap assessment. Two policies were developed for the fictional organization EduTrack Learning Solutions to strengthen information security governance and support the implementation of an Information Security Management System (ISMS).

## Repository Contents

- `information-security-policy.md` – Organization-wide Information Security Policy.
- `access-control-policy.md` – Access Control Policy focused on identity, authentication, and privileged access management.

## Policy Objectives

The policies were developed to:

- Establish a formal information security governance framework.
- Define clear security roles and responsibilities.
- Strengthen access control and authentication practices.
- Support continual improvement of the Information Security Management System (ISMS).
- Improve alignment with ISO/IEC 27001:2022 Annex A controls.

## ISO/IEC 27001:2022 Annex A Controls

### Information Security Policy

- A.5.1 – Policies for Information Security
- A.5.9 – Inventory of Information and Other Associated Assets
- A.5.23 – Information Security for Use of Cloud Services
- A.6.3 – Information Security Awareness, Education and Training
- A.6.8 – Information Security Event Reporting

### Access Control Policy

- A.8.2 – Privileged Access Rights
- A.8.5 – Secure Authentication
- A.6.5 – Responsibilities After Termination or Change of Employment
- A.8.15 – Logging
- A.8.16 – Monitoring Activities
